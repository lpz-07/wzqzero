# 观赛降噪助手 — WZQ Zero

> 一款 Chrome/Edge 浏览器扩展，基于实时谱减法降噪算法，智能抑制体育直播中的观众呐喊声，同时保留解说和击球声。

---

## 目录结构

```
wzqzero/
├── manifest.json         # Manifest V3 配置
├── background.js         # Service Worker：设置管理、tabCapture 协调
├── content.js            # 页面注入：媒体元素挂载、静音协调
├── audio-processor.js    # AudioWorklet：FFT 谱减法降噪核心
├── offscreen.html/js     # Offscreen Document：tabCapture 模式音频处理
├── popup/
│   ├── popup.html        # 控制面板 UI
│   └── popup.js          # 控制面板逻辑
├── icons/                # 扩展图标（16/48/128px）
└── README.md
```

## 安装步骤

### 方式一：开发者模式加载（推荐）

1. 打开 Chrome/Edge，访问 `chrome://extensions` 或 `edge://extensions`
2. 右上角开启 **"开发者模式"**
3. 点击 **"加载已解压的扩展程序"**，选择本项目根目录（含 `manifest.json` 的文件夹）
4. 扩展图标出现在浏览器工具栏，安装完成

### 方式二：打包安装

1. 在 `chrome://extensions` 页面点击 **"打包扩展程序"**
2. 选择根目录，生成 `.crx` 文件后拖拽安装

## 使用方法

### 快速上手（观看央视视频乒乓球直播）

1. 访问 [央视视频直播](https://www.yangshipin.cn/live/detail?pid=600220583)
2. 点击工具栏的扩展图标，弹出控制面板
3. 将 **音频捕获模式** 切换为 **"标签页捕获（通用）"**（推荐，兼容所有视频网站）
4. 开启 **"观众人声抑制"** 开关
5. 调整 **降噪强度** 滑块（默认 70%）
6. 选择预设模式：
   - 🏟️ **观赛模式**：抑制观众呐喊声，保留解说和击球声
   - 🎙️ **纯解说**：更激进抑制，尽量只保留解说
   - 🔊 **原声**：关闭降噪，恢复原始音频

### 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Alt+S` | 一键开关降噪 |

## 技术原理

### 音频捕获方案

插件支持两种音频捕获模式：

| 模式 | 原理 | 适用场景 |
|------|------|----------|
| 直接模式 | `createMediaElementSource` 劫持 `<video>` 元素 | 同源或允许 CORS 的视频 |
| 标签页捕获 | `chrome.tabCapture` + Offscreen Document | 所有网站（推荐） |

**标签页捕获流程：**
```
popup → background → tabCapture.getMediaStreamId()
                   → offscreen.createDocument()
                   → offscreen.js: getUserMedia(streamId)
                   → AudioContext → AudioWorklet → 扬声器
                   → 原标签页静音（tabs.update({muted:true})）
```

### FFT 谱减法降噪算法

核心处理器 `audio-processor.js` 实现了基于 STFT 的实时降噪：

```
输入音频 → [分帧加窗] → [FFT] → [噪声底噪估计] → [Wiener 增益]
        → [IFFT] → [OLA 叠加合成] → 输出音频
```

**关键参数：**
- FFT 帧长：512 样本（≈ 11.6 ms @ 44100 Hz）
- 帧移：128 样本（75% 重叠，OLA 归一化因子 = 2.0）
- 窗函数：Hann 窗
- 噪声估计：25 帧最小统计量法（≈ 370 ms 历史窗口）
- 增益函数：过减谱减 + Wiener 风格增益，并加入频域/时域平滑抑制“音乐噪声”

**频率选择性抑制：**
- 观赛模式：200–4000 Hz（人声核心频段）
- 纯解说模式：80–6000 Hz（更广频段）
- 人声范围外的低频（击球声冲击感）和高频透传不处理

**噪声底噪估计（最小统计量法）：**
```
noise_floor[k] = smooth(min(mag_history[k]))
```
观众呐喊声为持续性背景声，其频谱最小值即为噪声底噪估计，
通过 Wiener 增益公式将其从总信号中减去。

**抗“兹拉”伪影策略：**
- 频域平滑：对每个频点增益做邻域平滑，减少孤立窄带尖刺
- 时域平滑：对相邻帧增益做指数平滑，降低帧间抖动带来的爆裂感
- 瞬态保护：当解说/击球能量突增时抬高增益下限，避免语音被过度削弱

**乒乓球场景专项优化：**
- 女声“加油”抑制：对约 700–2800 Hz 频段施加更强抑制，重点压制观众口号声
- 击球声保护：通过谱通量瞬态检测，在球击台瞬间降低部分频段抑制，保留清脆击球感

**处理延迟：**  
FFT 帧缓冲延迟约 11.6 ms，OLA 延迟约 8.7 ms，总延迟 < 30 ms，
对直播观赛体验无影响。

## 已知限制

1. **跨域视频**：若视频源不支持 CORS，直接模式无法工作，请切换为标签页捕获模式
2. **视音频同步**：标签页捕获模式引入约 20–50 ms 额外延迟，在快节奏运动场景中基本不可感知
3. **音频分离精度**：本插件使用信号处理方法（非 AI 模型），无法完美区分解说与观众声；当两者同时出现时，会对两者都有一定程度的抑制
4. **平台兼容性**：适配 Chrome 116+、Edge 116+（需支持 Offscreen Document API）
